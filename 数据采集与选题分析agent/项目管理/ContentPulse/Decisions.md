# ContentPulse Decisions

## 已确认决策

### 1. 关键词与平台关系
- 一个关键词可以同时绑定多个平台
- 关键词配置项不是纯字符串，而是监控目标对象

### 2. 抓取触发时机
- 进入分类时不自动全量抓取
- 仅在以下两种情况触发抓取：
  - 新增关键词
  - 点击内容页一键更新

### 3. 内容数量上限
- 每个关键词 × 每个平台，最多保留 20 条内容
- 页面中所有数量展示统一基于可展示数据

### 4. 平台刷新逻辑
- 当前选中单个平台时，一键更新默认刷新当前平台 + 当前关键词
- 当前选中“全部平台”时，刷新全部已启用且已接入的平台

### 5. 小红书抓取窗口
- 默认抓取最近 7 天，而不是单日
- 原因：单日窗口会导致日期轴塌缩、内容量偏少

### 6. 当前已接入平台
- 公众号
- 小红书

### 7. 搜索历史设计
- 搜索历史作为一级页面，而不是抽屉或弹层
- 一级页展示历史记录列表
- 点击某条记录后展开查看该次搜索自己的内容快照与选题快照
- 每次搜索都应有独立历史，不允许被后一次搜索覆盖

### 8. 数据持久化策略
- 最新内容继续保存到 `collected_contents`
- 按查询归档的历史内容保存到 `search_query_contents`
- 选题快照按搜索记录保存到 SQLite

### 9. 项目管理方式
- 开发过程继续用 spec / plan / progress 的文档方式管理
- Obsidian 的主入口放在 `项目管理/ContentPulse/`
